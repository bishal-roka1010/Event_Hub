About this project

EventHub — ICT203 Assessment Build

A frontend application that delivers Home, Events Listing, Event Details (with RSVP), Registration, and a personal Dashboard — built with Bootstrap and vanilla JavaScript, using localStorage for demo persistence.

Work Distribution & Roles

We divided the project by strengths to keep velocity and quality high. Member A (Frontend Lead) set up the HTML structure, routing across pages, and events.json schema, client-side RSVP storage in localStorage. Member B (UI/UX) focused on visual design and interaction polish: responsive layouts, accessible semantics, skeleton loading states, and consistent patterns for toasts and modals. Both members contributed to testing and bug fixes.

Collaboration happened in small iterations: we groomed tasks together, kept a checklist for acceptance criteria, and performed lightweight code reviews before merging.

Project Highlights

The most exciting part is the end-to-end user flow that feels “real” even without a backend. The Home page includes a hero search with date pickers and categories, a featured event, live stats, and delightful touches (chips, ticker, countdown). The Events Listing supports grid/list views, keyword search, category/date filters, sorting, favorites, and pagination. On the Event Details page, users see a large poster, a map embed, a clear summary, an RSVP button with accessible feedback, and a small public demo attendee list.

The Dashboard consolidates a user’s registrations, adds exports (JSON/CSV) and calendar (.ics) downloads, and provides simple analytics (counts, sparkline, achievements). Across the app we prioritized accessibility: semantic headings, alt text, aria-live regions for dynamic updates, and safe HTML escaping to prevent injection. All of this is wrapped in a modern, responsive design that maintains performance on mobile.

Challenges Faced

Our biggest challenges were mostly integration details. First, getting sticky headers to play nicely with notifications — we initially had success toasts rendering under the navbar; we resolved this by positioning toasts below the measured navbar height. Second, date/time handling varied across browsers, so we normalized comparisons using ISO date strings and carefully built local Date objects for countdowns and calendar exports.

Finally, keeping paths consistent in a local Apache/XAMPP setup required attention to relative URLs and folder names to avoid 404s when moving between pages.
